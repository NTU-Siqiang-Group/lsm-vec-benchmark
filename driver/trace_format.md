# Shared streaming-trace format (the contract all systems consume)

Every system under test (ours, SPFresh, SPANN+, DiskANN) replays the **same** trace so the
insert/delete sequences are byte-identical across systems. One trace = one
`(dataset, scale, ratio)` cell. Generated by `bench/driver/gen_workload.py` (§4 of the runbook).

## Directory layout

```
work/<dataset>_<scale>_<ratio>/
  manifest.json          # run parameters (see schema below)
  base.fbin              # initial index vectors (the N base rows)
  base.ids.u32           # global ids assigned to base rows (default: 0 .. N-1)
  pool.fbin              # insertion source vectors (disjoint from base, ~0.5N rows)
  pool.ids.u32           # global ids for pool rows (default: N .. N+|pool|-1)
  query.fbin             # held-out queries, fixed for the whole run
  epoch_000.ins.u32      # global ids inserted during epoch 0 (rows pulled from pool by id)
  epoch_000.del.u32      # global ids deleted during epoch 0 (subset of currently-live ids)
  epoch_001.ins.u32
  epoch_001.del.u32
  ...
  gt/epoch_000.gt100     # groundtruth top-100 for query.fbin vs the ACTIVE set after epoch 0
  gt/epoch_010.gt100     # only at gt checkpoints (every G epochs)
  ...
```

`epoch_*.del.u32` may be empty (R-INS has no deletes — the file is present but zero-length,
or omitted; readers treat a missing del file as "no deletes this epoch").

## Binary file formats

All integers little-endian. These match DiskANN's `.fbin` / groundtruth conventions so the same
files feed DiskANN's tools directly.

### `.fbin` — vector matrix
```
int32   n        # number of rows
int32   d        # dimension
float32 data[n*d]  # row-major; row i = data[i*d .. i*d+d-1]
```
SIFT base vectors are uint8 in the source but are written here as float32 (the universal payload).
SPACEV likewise widened to float32. Distance is L2 for both datasets.

### `.u32` — id list
```
uint32  ids[]    # length = filesize/4; no header
```
Global ids are stable across the whole run, so a delete list unambiguously names live rows.

### `.gt100` — groundtruth (DiskANN `compute_groundtruth` format)
```
uint32  nq       # number of queries (must equal query.fbin n)
uint32  K        # neighbors per query (100)
uint32  ids[nq*K]    # nearest-neighbor global ids, row-major (closest first)
float32 dists[nq*K]  # corresponding squared-L2 distances (closest first)
```
Recall@10 uses the first 10 ids of each query's row. The harness only *measures* recall at gt
checkpoints; latency/throughput are measured every epoch.

## manifest.json schema

```json
{
  "dataset": "sift",
  "dim": 128,
  "metric": "l2",
  "scale": 1000000,
  "ratio": "r9010",
  "n_base": 1000000,
  "n_pool": 500000,
  "n_queries": 10000,
  "n_epochs": 50,
  "ops_per_epoch": 10000,
  "gt_interval": 10,
  "seed": 1,
  "delete_policy": "uniform_live"
}
```

`ops_per_epoch` is the per-epoch operation budget (= 1% of N). For R-INS all of it is inserts; for
R-9010 it splits 0.9%·N inserts + 0.1%·N deletes. The active-set size after epoch k is
`n_base + Σ_{0..k}|ins| − Σ_{0..k}|del|`; the harness asserts this every epoch.

## Determinism

`seed` is written into the manifest; the generator's RNG is seeded from it, so every system
consumes identical ins/del sequences. Re-running the generator with the same args reproduces the
trace bit-for-bit.
